An offline-first, professional personal book cataloguing application built using Flutter. **DRAFT** combines minimal aesthetics, robust SQLite-backed local persistence, full-text search (FTS5), and a fluid, therapeutic user interface.

---

## Features

- **Double-Layer Collections:** Separate shelves for **Owned** books and your **Wishlist**.
- **Categorization:** Multi-category tagging (up to 4 categories per book) designed to fit perfectly on catalog cards.
- **Therapeutic Interactions:** Custom, medium-paced subtle animations using `flutter_animate` alongside a GPU-optimized custom bottom shader fade in book details.
- **Fast Full-Text Search:** Direct SQLite FTS5 index integration allowing instantaneous local queries by title or author name.
- **Quick API Lookup:** Fast search integration with Google Books and Open Library APIs, backed by short connect timeouts, client-side fallback handling, and in-memory query caching.
- **Manual Sorting:** Rearrange collections dynamically via drag-and-drop powered by a sparse `REAL` positioning algorithm.
- **Local-First Reliability:** DB relational validation checks ensuring data integrity (e.g., preventing wishlist items from being marked as favorites).

## Architecture

```
  ┌──────────────────────────────────────────────────────────┐
  │                   UI (Widgets & Screens)                 │
  └────────────────────────────┬─────────────────────────────┘
                               │ Reads & writes via
  ┌────────────────────────────▼─────────────────────────────┐
  │     State Management (Riverpod StateNotifiers/Streams)   │
  └────────────────────────────┬─────────────────────────────┘
                               │ Invokes abstract contracts
  ┌────────────────────────────▼─────────────────────────────┐
  │                 Repository Interfaces                    │
  │     (Decouples widgets from underlying libraries)        │
  └────────────────────────────┬─────────────────────────────┘
                               │ Implemented by
  ┌────────────────────────────▼─────────────────────────────┐
  │         Data Layer (Drift ORM + SQLite Local DB)         │
  └──────────────────────────────────────────────────────────┘
```

- **Feature-Based Structure:** Code is organized by domain features (`features/library`, `features/book_detail`, `features/add_book`, `features/settings`) under `lib/features/`.
- **Mapping Boundary:** Domain models (`domain/models/`) are completely separated from database-specific entities generated by Drift. Entities are mapped cleanly at the repository implementation layer.

## Setup & Installation

### 1. Font Assets
Download the [Manrope Font Family](https://fonts.google.com/specimen/Manrope) and place the `.ttf` files under:
```
assets/fonts/Manrope-Regular.ttf
assets/fonts/Manrope-Medium.ttf
assets/fonts/Manrope-SemiBold.ttf
assets/fonts/Manrope-Bold.ttf
assets/fonts/Manrope-ExtraBold.ttf
```

### 2. Configure API Keys
To protect keys from version control leakages, create a file named `lib/api_keys.dart`:
```dart
const googleBooksApiKey = 'YOUR_GOOGLE_BOOKS_API_KEY';
```
Ensure you have restricted this key in the Google Cloud Console to your target package name and certificate fingerprint. (This file is ignored by `.gitignore`).

### 3. Dependencies & Code Generation
Run the following commands to configure dependencies and trigger the Drift builder:
```bash
flutter pub get
flutter pub run build_runner build --delete-conflicting-outputs
```
This triggers compilation of:
- `lib/data/local/database.g.dart` (Drift database initialization)
- `lib/data/local/daos/books_dao.g.dart`
- `lib/data/local/daos/categories_dao.g.dart`

### 4. Running the Application
To launch the app on your current connected device/simulator:
```bash
flutter run
```

## Design & Structural Decisions

- **UUID Primary Keys:** Promotes future conflict-free remote synchronization support.
- **Sparse Sort Ordering (`sort_order REAL`):** Sorting relies on fractional positioning, permitting list-sorting adjustments with a single update payload without index rewrites.
- **SQLite FTS5 Triggers:** Search indexes are kept in sync automatically via background database triggers on inserts, updates, and deletes.
- **No Accidental Swipe-Deletes:** Interactive deletion utilizes explicit long-press menus to mitigate accidental destruction of database items.
- **Safe-Start Housekeeping:** Soft-deleted records are garbage collected post-first-frame rendering, keeping slow disk I/O outside the cold-start critical path.
